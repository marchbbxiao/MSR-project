1. conda 虛擬環境建立

在現有的 (base) Anaconda 環境下，建立獨立的 openmc 虛擬環境
Python 版本：3.11
指令：conda create -n openmc python=3.11
理由：隔離套件相依性，避免環境汙染

2. OpenMC 安裝

來源：conda-forge 頻道
版本：OpenMC 0.15.3
指令：conda install -c conda-forge openmc
驗證：python -c "import openmc; print(openmc.__version__)"

3. 截面資料庫下載與設定
⚠️ 與鄧永宏不一致的說明：
鄧永宏使用 ENDF/B-VII.1，本專案目前使用 ENDF/B-VIII.0。
原因如下：

原本計劃下載 VII.1，但第一次下載時使用了錯誤的連結，下載到 VIII.0
查閱 BNL 驗證報告後確認：兩個版本對 MSRE keff 的影響只有約 250 pcm，差異很小
在階段一（MSRE keff 驗證），目標是確認工具鏈正確運作，不是精確複刻數值，因此 VIII.0 完全可用
預期算出的 keff ≈ 1.022（對比實驗值 0.99978），這個系統性偏差是已知的 MSRE 模型問題，與資料庫版本無關

後續行動：進入階段二（MSFR）時，需要換回 ENDF/B-VII.1 以與鄧永宏的設定一致。屆時下載連結為：
https://anl.box.com/shared/static/9igk353zpy8fn9ttvtrqgzvw1vtejoz6.xz
截面資料庫路徑：
/home/bruce/nuclear_data/endfb-viii.0-hdf5/cross_sections.xml
環境變數設定：
bashecho 'export OPENMC_CROSS_SECTIONS=~/nuclear_data/endfb-viii.0-hdf5/cross_sections.xml' >> ~/.bashrc
