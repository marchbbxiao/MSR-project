## 階段四：Continuation Method（延續法）（11:00-14:00）

### 策略
Gemini 建議：延續法（Reynolds Number Ramping），從低速出發，逐步升速至全載。
物理意義：讓壓力場和速度場有時間協同發展，避免高 Re 對流項直接衝擊。

**關鍵警告（Gemini 提醒）**：k 和 ε 的入口 BC 必須隨速度同步縮放！
- k = 1.5 × (I × U_in)²，I=5%
- ε = Cμ^0.75 × k^1.5 / L，L=0.1m，Cμ=0.09

### th_run1.i（U=0.01 m/s，Re≈3,800）
- k = 3.75e-7，ε = 3.87e-10
- 結果：iter 1-60 壓力有下降趨勢（1e-1 → 1e-3）✅ **首次看到壓力真正收斂**
- iter 63 TKED 殘差跳到 0.99 → 爆炸
- **診斷**：U=0.01 m/s 對應 Re=3,800，接近層流，k-ε 是 High-Re Model，在此 Re 下 ε²/k 發散（除以零危機）
- Gemini 建議：起步速度太低，需提高到 U=0.1 m/s（Re≈38,000）

### th_run2.i（U=0.1 m/s，Re≈38,000）
- k = 3.75e-4，ε = 3.87e-6
- 加入 lower_bound（k: 1e-10，ε: 1e-12）和 PETSc VI（vinewtonrsls）
  - 注意：這些參數在 SIMPLENonlinearAssembly 中是 unused parameter，未生效
- 結果：TKE 在 iter 60 降到 ~1e-4，TKED 到 ~1e-5 ✅ **湍流方程式首次真正收斂**
- 壓力 iter 67 降到 ~1e-8 ✅ **歷史最佳**
- 動量在 iter 50 後反覆爆炸
- **新診斷**：動量殘差 ~0.333 週期性出現，壓力與動量脫鉤

### th_run3.i（momentum relaxation=0.3，num_iterations=1000）

## 階段五：策略 B - Transient 求解器（14:00-18:00）

### 目標
切換到暫態求解器，讓時間導數項 ∂(ρv)/∂t 提供「慣性彈性」穩定矩陣。

### th_transient.i 建立過程
從 `th_run4b.i` 複製並進行以下修改：

**1. Mesh 設定**
```
file = th_run4b_out.e
use_for_exodus_restart = true
```

**2. Variables 加入初始場讀取**
```
initial_from_file_var = vel_x/y/z/pressure/TKE/TKED
initial_from_file_timestep = 1
```
（注意：LAST 不被支援，需用數字 1）

**3. AuxVariables 加入 mu_t 初始場**
`th_run4b_out.e` 包含的變數：TKE, TKED, mu_t, power_density, pressure, vel_x, vel_y, vel_z, yplus
未讀取 mu_t 導致初始 NaN，後來加入：
```
initial_from_file_var = mu_t
```

**4. 移除 SIMPLE 專用設定**
- `nl_sys_names` → 移除
- `solver_sys` → 移除（全部 6 個變數）
- `previous_nl_solution_required = true` → 改為保留（Newton 需要）
- `p_diffusion`（FVAnisotropicDiffusion）和 `p_source`（FVDivergence）→ 替換為 `INSFVMassAdvection`
- `pressure_tag` 和 `extra_vector_tags` → 注解掉
- `pressure_gradient_tag` → 移除

**5. 加入時間項 kernels**
```
[vel_x_time] type = INSFVMomentumTimeDerivative ...
[vel_y_time] type = INSFVMomentumTimeDerivative ...
[vel_z_time] type = INSFVMomentumTimeDerivative ...
```

**6. Rhie-Chow 插值器**
從 `INSFVRhieChowInterpolatorSegregated` → `INSFVRhieChowInterpolator`（已確認正確）

**7. Executioner 改為 Transient**
嘗試過：
- `solve_type = NEWTON` + `lu` → 記憶體爆炸（211,116 DOF 全耦合，需數百GB）
- `solve_type = NEWTON` + `hypre boomeramg` → 初始殘差 6004，Newton 發散
- `solve_type = PJFNK` + `asm ilu` → 同樣 6004，Newton 方向錯誤

### 卡關診斷
**關鍵現象**：初始殘差永遠是 **6.004574e+03**，不隨 dt 改變（已試到 1e-12）

**Gemini 解釋**（確認）：
- Time step 0 時，Newton 評估 φ^old（exodus 讀入的場）代入 Newton 框架的 L(φ)
- 時間導數項 (φ^old - φ^old)/Δt = 0，所以不隨 dt 改變
- 6004 代表 SIMPLE 的壓力場（Poisson 修正框架）不滿足 Newton 的連續方程式（INSFVMassAdvection）
- 這是「水土不服」問題，不是數值問題
