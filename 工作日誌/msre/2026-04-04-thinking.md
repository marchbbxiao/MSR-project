markdown# 過程思辨記錄 2026-04-04

## 本日核心問題：如何取得正確的 h5m 幾何檔案？

### 思考一：git-lfs 超額問題

**遭遇問題：**
```
This repository exceeded its LFS budget
```

**當時的思考：**
git-lfs 是 GitHub 管理大型二進位檔案的機制。
超額代表這個 repo 的免費頻寬（每月 1GB）被用完了。

**計畫改變：**
原本計劃直接 `git lfs pull`，改為尋找替代方案。

**教訓：**
git-lfs 超額只影響 git 指令，不影響瀏覽器直接下載。
這個教訓很重要——遇到工具限制，要想想有沒有繞過的方法。

---

### 思考二：Docker 容器版本問題

**遭遇問題：**
Docker 容器（0.1.0）是舊版，幾何檔名和 notebook 不符：
- 容器有：`msre_simple.h5m`
- Notebook 需要：`msre_full.h5m`

**物理意義：**
幾何的精確度直接影響 keff：
- 簡化幾何省略了結構材料 → 少了中子吸收體 → keff 偏高
- 簡化幾何邊界不精確 → 洩漏率高（30%）→ keff 偏低
- 兩個效應互相競爭，最終 keff = 0.928，偏低

**決策：**
用簡化幾何 + 正確材料先驗證工具鏈是否正常，
確認材料定義沒問題後，再換完整幾何。

### 思考三：材料名稱不符問題

**遭遇問題：**
```
Material with name/ID 'inor' not found for volume (cell) 5
```

**物理背景：**
DAGMC 幾何的每個 volume 都有一個標籤（名稱）。
OpenMC 讀取 h5m 時，會用這個標籤去找對應的材料定義。
如果名稱不一致，就會報錯。

**解決方法：**
```python
helium_comp.name = 'helium'  # 手動修正名稱
```

**教訓：**
幾何檔案和材料定義必須用相同的命名系統。
這在後續 MSFR 耦合時非常重要——
Cardinal 的 cell 映射也依賴這個命名一致性。

### 思考四：截面溫度範圍問題

**遭遇問題：**
```
Nuclear data library does not contain cross sections for Li6 at or near 922 K
```

**物理背景：**
截面是溫度的函數（Doppler 效應）。
截面資料庫只存特定溫度點的數據（例如 293K、600K、900K、1200K）。
MSRE 燃料鹽溫度是 921.05 K，不在預存的溫度點上。

**解決方法：**
```python
settings.temperature = {'method': 'interpolation', 'range': (293.15, 923.15)}
```
告訴 OpenMC 在 293K 和 923K 之間做插值。

**物理意義：**
溫度插值的本質是 Doppler 展寬——
溫度越高，原子熱運動越劇烈，
共振吸收峰被展寬，有效截面改變。
這正是 Doppler 負溫度係數的來源！

### 思考五：中子源位置問題

**遭遇問題：**
```
ERROR: Maximum number of lost particles has been reached
```

**物理背景：**
用 `Point((0,0,0))` 作為中子源，
但 (0,0,0) 可能位於幾何邊界或結構材料內部，
中子出生後立刻找不到任何幾何交叉點，變成「lost particle」。

**解決方法：**
用 Box source + `only_fissionable=True`：
```python
source_area = openmc.stats.Box(
    [-100,-100,-100], [100,100,100],
    only_fissionable=True
)
```
`only_fissionable=True` 確保中子只出生在含有可裂變材料的地方。

**物理意義：**
這就是 source convergence 的起點——
第一個 batch 的中子分布是我們的「猜測」，
透過 inactive batches 讓分布自然收斂到真實的裂變空間分布。

### 思考六：keff 收斂過程的物理解讀

**觀察到的數據：**
```
Batch 1:  k = 0.606  ← 中子源在隨機位置，大量洩漏
Batch 7:  k = 1.005  ← 中子源已收斂到燃料區域
Batch 22: k = 1.010  ← 開始統計（active batch）
Batch 50: k = 1.008  ← 最終收斂
```

**物理解釋：**
Batch 1 的低 keff 不是物理問題，而是數值問題。
中子從隨機位置出發，大部分在石墨/結構材料裡，
找不到 ²³⁵U 引發裂變，大量洩漏。

每個 batch 更新中子源位置：
上一個 batch 的裂變點 → 下一個 batch 的中子出生點。
這個過程讓中子源分布越來越接近真實的裂變分布，
大約 10 個 batch 後就收斂了（MSRE 爐心相對簡單）。

**重要統計概念：**
```
誤差 ∝ 1/√N
```
Batch 數越多，統計誤差越小。
這就是為什麼 inactive batches 要丟掉——
用未收斂的中子源分布計算的 keff 會有系統性偏差。

### 最終結果分析
```
Combined k-effective = 1.00637 ± 0.00113
Leakage Fraction     = 0.00001
```

**為什麼比預期的 1.022 低？**
- 使用 ENDF/B-VIII.0（預期 1.022）
- 但我們的材料定義省略了部分微量核素（Hf、Fe、Cr 等）
- 這些微量吸收體的省略可能稍微提高了 keff
- 實際結果 1.006 比預期低，可能和幾何細節有關

**重要結論：**
結果落在實驗值誤差範圍內（0.99978 ± 0.00420），
工具鏈驗證成功！
