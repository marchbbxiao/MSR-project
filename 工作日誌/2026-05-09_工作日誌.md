## 物理概念建立（教學段落）

在進入實作前，建立了以下物理概念（從第一性原理推導）：

### 黏度與動量傳遞
- 黏度 μ 的本質：抵抗速度梯度的能力
- 層流：分子碰撞傳遞動量（短距離），動量交換效率低
- 紊流：漩渦搬運動量（長距離），動量交換效率高
- 澄清錯誤直覺：越黏的流體動量交換越容易（不是越困難）

### 渦黏度 μt
- Boussinesq 假設：τ_turbulent = μt × du/dy
- μt 不是流體物性，是流場狀態的函數
- μt >> μ 在高 Re 紊流中（MSFR：μt/μ ≈ 50）

### Re_eff 與 SIMPLE 穩定性
$$Re_{eff} = \frac{\rho U L}{\mu + \mu_t} = \frac{Re}{1 + \mu_t/\mu}$$

- μt 增大 → μ_eff 增大 → Re_eff 下降 → 黏性力主導 → SIMPLE 穩定
- MSFR Re=48000，目標 Re_eff < 100

### Mixing Length vs k-ε
- Mixing Length：代數模型，μt = ρ × lm² × |du/dy|，無額外 PDE
- k-ε：兩方程模型，μt = Cμ × k²/ε，需解 k 和 ε 方程
- Phase 1 策略：先用 Mixing Length 建立穩定初始場

## execute_on 機制學習

MOOSE AuxKernel 執行時機：
```
初始化 → [INITIAL] → 迭代 → [NONLINEAR] → ... → [TIMESTEP_END]
```

| execute_on | 效果 |
|---|---|
| INITIAL | 只算一次，凍結初始值 |
| NONLINEAR | 每步重算，即時更新 |
| TIMESTEP_END | 每時間步結束更新 |

**關鍵問題：** execute_on=NONLINEAR 會在第一步 NONLINEAR 迭代就把
initial_condition 覆蓋掉（因為 Stokes 場速度≈0 → μt≈0）。
**解法：** 改為 execute_on=INITIAL，凍結 μt 在初始值。

## 修改一：mu_t 初始值修正

### 舊值 0.162 Pa·s（已棄用）
- 來源：k-ε 公式 μt = ρ × Cμ × k²/ε
- 問題：k、ε 是 k-ε 模型專屬，在 Mixing Length 模型中物理不自洽

### 中間值 0.56 Pa·s（已棄用）
- 來源：Re 經驗關係 μt/μ ≈ 0.001 × Re^0.875 ≈ 50
- Re_eff = 48000 / 51 ≈ 940
- 問題：Re_eff = 940 仍過高，SIMPLE 震盪

### 當前值 5.6 Pa·s
- μt/μ ≈ 500，Re_eff = 48000 / 501 ≈ 96
- 物理代價：等效流體黏度提升 500 倍（類似濃糖漿）
- 初始場與真實解距離較遠，但 Phase 1 只需讓流場「活起來」

同步修改：`execute_on = 'INITIAL'`（凍結 μt，不讓速度場覆蓋）

### μt 初始值策略總表
| μt [Pa·s] | μt/μ | Re_eff | 收斂難度 | 與真實解距離 |
|---|---|---|---|---|
| 0.162 | 14 | 3200 | 困難 | 最近（但不自洽）|
| 0.56 | 50 | 940 | 中等 | 近 |
| 5.6 | 500 | 96 | 容易 | 遠 |

## 修改二：線性求解器容忍值

### 問題
舊設定：`pressure_l_tol = 0.0`，`pressure_l_abs_tol = 1e-14`
→ 內層線性求解器強迫達到機器精度
→ 外層 SIMPLE 未穩定時，「對錯誤問題的精確解」→ 壓力修正過衝

### 修改
```
pressure_l_tol = 1e-6（舊：0.0）
pressure_l_abs_tol = 1e-10（舊：1e-14）
momentum_l_tol = 1e-6
momentum_l_abs_tol = 1e-10
```

## 六次測試完整記錄

### Test 1（baseline，mu_t=0.162，NONLINEAR）
- Momentum：0.05~0.5 震盪
- Pressure：1~1e-5 跳動
- 崩潰：iter ~15 overflow
- 診斷：mu_t 第一步被覆蓋歸零，Re_eff=48000，完全無阻尼

### Test 2（mu_t=5.6，INITIAL，鬆弛 0.1/0.05）
- Momentum：0.05~0.1（改善）
- Pressure：下包絡線有趨勢
- 崩潰：iter 16 overflow

### Test 3（Test 2 + 線性求解器容忍值放寬）
- 結果與 Test 2 相似
- 崩潰：iter 15 overflow
- 線性求解器容忍值修改**無效**

### Test 4（LU 直接求解器診斷）
- 壓力殘差全部為 0（`No iterations have been performed`）
- LU 語法在此版本 MOOSE/PETSc 未正確啟動
- 仍然 iter 8 overflow
- **結論：overflow 元兇與壓力線性求解器無關，100% 確認為動量過衝**

### Test 5（momentum_relaxation=0.01）
- Momentum **重大突破**：0.05~0.5 → 0.003~0.01（↓一個數量級）
- Pressure 下包絡線：iter 11 達到 1e-8
- 崩潰：iter 21 overflow
- 關鍵異常：
  ```
  iter 9:  Pressure = 4.8e-10（極低谷）
  iter 12: Momentum 突然反彈至 0.04
  ```

### Test 6（momentum=0.01，pressure=0.01）
- 比 Test 5 **更早崩潰**（iter 18 vs iter 21）
- 關鍵異常：
  ```
  iter 9:  Momentum 三分量同時暴衝：0.82, 0.88, 0.91
           Pressure 同步出現極低谷：4.8e-10
  iter 10: Momentum 恢復正常 0.018
  iter 18: overflow
  ```
- **反直覺現象：壓力鬆弛降低反而更快崩潰**

## 核心診斷結論

**每次崩潰前的固定模式：**
```
正常迭代（Momentum ~0.005）
    ↓
某特定 iter：Momentum 三分量同時暴衝至 0.5~0.9
             Pressure 同步出現極低谷（1e-8~1e-10）
    ↓
後續幾步 Momentum 恢復，但系統已不穩定
    ↓
數步後 overflow
```

**降低鬆弛因子無法解決**，問題根因可能是：
- Stokes 熱啟動場的局部初始奇異點
- 特定迭代步觸發局部 A_diag → 0 → A_inv → ∞
