## 三、人工阻尼實作（th_phase1_from_th.i）

### 設計原理
在動量方程右側加體積力 F_body：
- 當 |v| > 5.0 m/s：F_body = -1e5 × v（巨大反向阻力）
- 當 |v| ≤ 5.0 m/s：F_body = 0（不干擾正常流場）

目的：防止 Sliver element 速度暴衝導致 A_diag → 0 → A_inv → ∞

### 實作步驟

**Step 1：[FunctorMaterials] 加入三個 ADParsedFunctorMaterial**
```
[drag_force_x]
  type = ADParsedFunctorMaterial
  property_name = 'drag_force_x'
  expression = 'vel_mag := sqrt(vel_x^2+vel_y^2+vel_z^2); if(vel_mag>5.0,-1e5*vel_x,0.0)'
  functor_names = 'vel_x vel_y vel_z'
[]
```
（y/z 同理）

**Step 2：[FVKernels] 加入三個 INSFVBodyForce**
```
[rubber_wall_x]
  type = INSFVBodyForce
  variable = vel_x
  functor = drag_force_x
  rhie_chow_user_object = 'rc'
  momentum_component = 'x'
[]
```

### 除錯歷程

| 嘗試 | 錯誤訊息 | 解決方法 |
|------|----------|----------|
| ParsedFunctorMaterial | `requesting vel_x as non-AD functor` | 換 ADParsedFunctorMaterial |
| use_ad = true | `unused parameter` | 此參數無效，改換 type |
| INSFVBodyForce | `missing momentum_component` | 加 `momentum_component = 'x/y/z'` |

### 結果（th_damp1.log）
- ✅ overflow 消失，速度不再暴衝到 1e+45
- ❌ 仍在 iter 14 崩潰
- **失敗模式改變：從「炸掉」→「Flip-Flop 震盪」**
- Pressure residual 在 1.0 和 1e-2 之間劇烈跳動

## 四、Gemini 諮詢（物理策略分析）

### 諮詢內容
提交完整診斷報告，詢問三個核心問題：
1. Transient 初始殘差 6004.574 的根本原因
2. MSFR 是否真的沒有穩態解
3. Flip-Flop 震盪的下一步策略

### Gemini 回覆摘要

**問題1：6004.574 = ρ·g 的 L2 norm**
- `4147 × 9.8 ≈ 4064`，L2 norm 就是這個量級
- 根本原因：速度量級 ~1，壓力量級 ~1e5，Jacobian 條件數爆炸
- **修法：pressure 變數加 `scaling = 1e-4`**

**問題2：MSFR 無穩態解——確認**
- Re≈446000 球形爐心噴流場 → 物理上是統計穩態（Limit cycle）
- Tano et al. 2025 本人也用 Pseudo-Transient / URANS
- SIMPLE 穩態求解是錯誤工具

**問題3：切換 Transient Newton**
- 優先策略：Transient + Newton + FieldSplit 預條件器
- 壓力 scaling 是 Newton 收斂的關鍵
- 時間平均後的 T(r)、v(r) 才是送給 OpenMC 的正確數據

## 六、Transient Newton 求解器建立（th_transient_v2.i）

### 從 SIMPLE 切換的六項關鍵修改

| 項目 | SIMPLE（舊）| Transient Newton（新）|
|------|------------|----------------------|
| 全域變數 | `pressure_tag = "pressure_grad"` | 移除 |
| [Problem] | `nl_sys_names` 四系統 + `previous_nl_solution_required` | 移除，單一系統 |
| [UserObjects] | `INSFVRhieChowInterpolatorSegregated` | `INSFVRhieChowInterpolator` |
| [Variables] | 每個變數有 `solver_sys = x_system` | 移除，加 `scaling=1e-4` 於 pressure |
| [FVKernels] 壓力 | `p_diffusion` + `p_source`（Ainv/HbyA）| `INSFVMassAdvection` |
| [FVKernels] 時間項 | 全部註解（SIMPLE 不允許）| 啟用 `INSFVMomentumTimeDerivative` x3 |
| [Executioner] | `SIMPLENonlinearAssembly` + relaxation | `Transient` + Newton + LU |

### Executioner 設定
```
type = Transient
solve_type = 'NEWTON'
petsc_options_iname = '-pc_type -pc_factor_mat_solver_type'
petsc_options_value = 'lu superlu_dist'
dt = 1e-3
end_time = 30.0
[TimeStepper]
  type = IterationAdaptiveDT
  optimal_iterations = 10
  cutback_factor = 0.5
  growth_factor = 1.2
[]
```

### 除錯歷程

| 嘗試 | 錯誤訊息 | 解決方法 |
|------|----------|----------|
| 第一次跑 | `No functor 'Ainv' requested by p_diffusion` | 移除 p_diffusion、p_source |
| 第二次跑 | `pressure lacks active kernel` | 加入 INSFVMassAdvection |
| FieldSplit | `must have at least two fields, not 0` | 改用 hypre（暫時）|
| hypre | Linear |R| 幾乎不動（5994 → 5928，60次迭代）| 改用 LU（superlu_dist）|

## 九、關鍵學習

1. **SIMPLE vs Newton 的根本差異**
   - SIMPLE：分離式，速度壓力輪流求解，需要 relaxation factor
   - Newton：全耦合，一個大 Jacobian，需要 scaling 和好的預條件器

2. **不可壓縮流 kernel 對應關係**
   - SIMPLE 壓力：p_diffusion + p_source（Ainv、HbyA functor）
   - Newton 壓力：INSFVMassAdvection（∇·v = 0）

3. **Pressure Scaling 的必要性**
   - 速度量級 ~1，壓力量級 ~1e5
   - 不加 scaling → Jacobian 條件數爆炸 → Newton 無法收斂
   - scaling = 1e-4 讓兩者量級一致

4. **FieldSplit 需要 MOOSE [Preconditioning] block**
   - 只靠 petsc_options 不夠
   - 需要明確告訴 MOOSE 哪些變數屬於哪個 field

5. **物理認識：MSFR 是統計穩態，非真穩態**
   - Re≈446000 噴流 → 渦旋脫落 → Limit cycle
   - 正確做法：Transient URANS + 時間平均
