## 上午：物理概念建立

在進入實作前，建立以下概念：

### αm + αp ≈ 1 的耦合原理

SIMPLE 的速度修正：v_new = v* + αm × A_inv × ∇p'
SIMPLE 的壓力修正：p_new = p* + αp × p'

αm 和 αp 必須滿足耦合一致性條件：αm + αp ≈ 1

崩潰邏輯：
```
αp 太小
→ 壓力修正不足 → 質量誤差累積（∇·v ≠ 0）
→ 速度場在無壓力約束下自由發展
→ 局部速度出現非物理極端值
→ 對流項離散化出現數值問題
→ A_diag → 0 → A_inv → ∞ → overflow
```

Test 6 比 Test 5 更早崩潰的原因：
αp 更小 → 質量誤差累積更快 → A_diag 更早趨近於零

### 凹角奇異點的物理機制

A_diag ∝ 面積/體積

在入口通道與球形壁面的垂直交界處（凹角），網格被切成極扁的四面體：
- 體積極小 → A_diag → 0 → A_inv → ∞
- 在 FVM 計算通量時：通量 = 速度差/網格體積 → 分母趨近零 → 爆炸

## 測試歷程

### Test 7（momentum=0.5, pressure=0.3, αm+αp=0.8）

主要變更：
- 回歸合理耦合值 αm+αp≈1
- 加入速度截斷 lower_bound/upper_bound

結果：
- 速度截斷語法不支援（unused parameter）
- Momentum 退步至 0.2~0.8（αm=0.5 太大）

教訓：αm+αp≈1 是必要條件但非充分條件，αm 本身也不能太大

主要變更：

- iter 15 Momentum 暴衝數值完全相同：0.994, 0.999, 0.969
- Ramping 無法避開 iter 15 的固定爆炸點

### Test 9（Test 8 + Postprocessors TIMESTEP_END）

主要變更：
- Postprocessors execute_on 從 NONLINEAR 改為 TIMESTEP_END

結果：
- 仍只輸出 time=0（SIMPLENonlinearAssembly 與標準輸出系統不相容）
- 殘差與 Test 8 完全相同

## ParaView 驗屍：成功定位奇異點 🎯

### 操作步驟
1. 安裝 ParaView 6.1.0（已預裝）
2. File → Open → `th_phase1_from_th_exodus_out.e`
3. 顯示變數：vel → Magnitude
4. 發現最大速度：**6.7e+45 m/s**（天文數字）
5. Filters → Threshold，vel > 1e+10，All Scalars 取消
6. 成功隔離出單一紅色網格

### 診斷結果

**只有一個奇異點：**
- 位置：爐心側面壁面中間高度
- 具體位置：入口通道（gap_enter_boundary）切割球形壁面（wall）的垂直溝槽交界處
- 速度值：6.7e+45 m/s（完全非物理）
- 類型：凹角（Re-entrant Corner）奇異點

**為什麼只有這一個爆炸（雖然類似凹角有多個）：**
- A_diag ∝ 面積/體積
- 這個特定網格的體積/面積比最極端（最薄的 Sliver element）
- 鄰近的凹角都有小體積，但這個網格的畸形程度特別嚴重

## Gemini 第四次諮詢

Gemini 確認這是 CFD 中的經典「凹角奇異點」問題。

### 治本方案（Cubit 幾何倒角）

在 Cubit 中對入口通道與球壁的交界邊緣施加小半徑倒角（Fillet），消除數學上的無限大梯度奇異性。同時檢查網格品質，砍掉 Aspect Ratio > 50~100 的 Sliver elements。

### 治標方案（ParsedFunctorMaterial 人工阻尼）

在 FVKernels 加入只針對超速網格生效的巨大反向阻力：

```
vel_mag > 5.0 → drag = -1e5 × velocity
vel_mag ≤ 5.0 → drag = 0
```
