## 關鍵學習

### 1. 如何讀 MOOSE 錯誤訊息
- `*** ERROR ***`：致命，有行號，直接看
- `*** Warning ***`：非致命，但 `unused parameter` 代表參數放錯位置
- `DIVERGED_LINE_SEARCH`：Newton 步過大，dt 會自動切半重試
- `DIVERGED_MAX_IT`：殘差平台，不是方向錯而是步長太小無法推進

### 2. Git 工作流習慣建立
- 每次有意義的改動立刻 commit
- Commit message 記錄「做了什麼、為什麼、結果是什麼」
- 不在程式碼裡留歷史殘骸，靠 Git 保存

### 3. [Physics] 系統的 block 結構
```
[Physics/NavierStokes/Flow/flow_physics]       → 速度、壓力
[Physics/NavierStokes/FluidHeatTransfer/...]   → 溫度
[Physics/NavierStokes/Turbulence/turb_physics] → TKE, TKED, μ_t
```
**耦合參數必須放在「使用方」的 block 裡，不是「被引用方」。**

### 4. Pseudo-Transient 的物理意義
∂(ρu)/∂t 項在矩陣對角線加入 ρV_c/Δt：
- 小 Δt：對角優勢強，穩定 Sliver 問題
- 大 Δt：趨近穩態解
- 這是解決非結構網格病態矩陣的標準工程手法
