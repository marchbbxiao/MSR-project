## 五、2026-05-13 晚：重大技術決策——轉向 MOOSE [Physics] 系統

### 決策背景

Gemini Deep Research 報告指出：
1. 手動 `[FVKernels]` 缺乏 skewness-corrected、Rhie-Chow 等穩定機制
2. SIMPLE 穩態無法應對 Sliver Elements（矩陣失去對角優勢）
3. MOOSE 2024/2025 已有完整的 `[Physics]` 系統，一行啟用 k-ω SST

### 為什麼 [Physics] + Pseudo-Transient 能解決問題

| 機制 | 作用 |
|------|------|
| skewness-corrected 面插值 | 消除 Sliver 幾何偏斜導致的梯度估算錯誤 |
| Rhie-Chow 體積力修正 | 防止同位網格的壓力震盪 |
| upwind 梯度限制器 | 防止局部速度過衝，保持矩陣正定性 |
| **Pseudo-Transient 時間導數** | **ρV_C/Δt 加到對角線，強制矩陣對角佔優** |

**Pseudo-Transient 的關鍵原理：**
```
穩態求解：矩陣無時間項 → Sliver 使對角線→0 → 矩陣病態 → 速度飆 10^6 m/s
暫態求解：∂(ρu)/∂t → 對角線加載 ρV_C/Δt
→ 即使 Sliver 使非對角線係數異常放大
→ 只要 Δt 足夠小，對角線就能壓制矩陣，速度不再發散
```

### 新架構設計（MOOSE 2024/2025 語法）

```ini
[Physics]
  [Flow]
    [flow_physics]
      compressibility = 'incompressible'
      momentum_advection_interpolation = 'upwind'
      pressure_face_interpolation = 'skewness-corrected'
      velocity_interpolation = 'rc'
      # 邊界條件...
    []
  []
  [FluidHeatTransfer]
    [energy_physics]
      energy_advection_interpolation = 'upwind'
      energy_face_interpolation = 'skewness-corrected'
    []
  []
  [Turbulence]
    [sst_physics]
      turbulence_handling = 'k-omega-sst'  # 一行取代所有手動 k-ε kernels
    []
  []
[]

[Executioner]
  type = Transient                # Pseudo-Transient（關鍵！）
  solve_type = 'NEWTON'
  petsc_options_iname = '-pc_type -pc_factor_mat_solver_type
                          -pc_factor_shift_type -snes_linesearch_type'
  petsc_options_value  = 'lu     mumps
                          NONZERO               bt'
  [TimeStepper]
    type = IterationAdaptiveDT
    dt = 1e-4                     # 極小初始步長，龐大 ρV_C/Δt 壓制初始衝擊
    cutback_factor = 0.5
    growth_factor = 1.2
    dtmax = 1.0
  []
  steady_state_detection = true
  steady_state_tolerance = 1e-6
[]
```

**優點：**
- 原本 945 行的 th_run4b_energy.i 縮短到 ~100 行
- k-ω SST 一行設定（取代 Tano 2022 建議的 Realizable k-ε）
- 自動生成標準變數（vel_x, vel_y, vel_z, pressure, T_fluid），
  Cardinal Transfer 直接對應
