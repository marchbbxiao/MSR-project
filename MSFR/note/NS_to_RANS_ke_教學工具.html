控制體積（Control Volume）是所有方程式的起點

想像燃料鹽流場中一個微小的正方體（dx × dy × dz）。所有守恆定律都在問同一個問題：

「這個小方塊內，某個物理量（質量/動量/能量）怎麼變化？」

所有方程式的共同骨架：時間項 + 對流項 = 擴散項 + 源項
差別只在「守恆的是什麼」以及「源項長什麼樣子」。
這個骨架就是 MOOSE 框架的根基，每個 FVKernel 對應一項。
Navier-Stokes 動量方程式（原始版）

1822年建立，描述所有黏性流體的瞬時運動。對層流、湍流、穩態、暫態都成立。點擊各項查看說明：

x 方向動量守恆（牛頓第二定律 F=ma 的連續體版本）
ρ ∂v/∂t  +  ρ (v·∇)v  =  −∇p  +  μ ∇²v  +  ρg
ρ ∂v/∂t — 時間加速度項（慣性項）
這塊流體動量隨時間的變化率。牛頓第二定律 F=ma 裡的「ma」就是這項。
穩態時理論上 = 0，但 SIMPLE 迭代過程仍需要它來組裝矩陣。
th.i → INSFVMomentumTimeDerivative
ρ (v·∇)v — 對流項（Advection）
流體把動量從 A 點「帶」到 B 點。這是非線性項（速度 × 速度梯度），也是 N-S 數值求解困難的主因。
Re 越高，這項相對黏性項越重要 → 湍流越激烈。
th.i → INSFVMomentumAdvection
−∇p — 壓力梯度項
壓力差驅動流體從高壓流向低壓。這就是為什麼 SIMPLE 演算法能用壓力來修正速度：
動量方程式裡有 −∇p，所以修改 p 就能改變 v，進而滿足 ∇·v=0（質量守恆）。
th.i → INSFVMomentumPressure
μ ∇²v — 黏性擴散項
動量透過分子黏性從快速流體傳到慢速流體（像「內部摩擦」）。
μ = 0.011266 Pa·s（燃料鹽分子黏度）。
⚠️ RANS 版本中這項變成 ∇·[(μ+μt)∇v̄]，加入湍流黏度 μt。
th.i → INSFVMomentumDiffusion（目前只有 μ，待補 μt）
ρg — 重力體積力
重力對每單位體積流體的作用力。
MSFR 球形爐心：z 方向 g = −9.81 m/s²，x/y 方向為零。
th.i → INSFVMomentumGravity（僅 z 方向）gravity='0 0 -9.81'

RANS：Reynolds 時間平均的推導過程

把所有量拆成「時間平均 + 瞬時擾動」：

v  =  v̄  +  v'      p  =  p̄  +  p'
時間平均值（我們求解的）
瞬時擾動（被打包掉）
對 N-S 做時間平均後得到 RANS

點擊各項查看與原始 N-S 的差異：

ρ ∂v̄/∂t  +  ρ (v̄·∇)v̄  =  −∇p̄  +  ∇·[(μ+μt)∇v̄]  +  ρg
ρ ∂v̄/∂t — 平均速度時間項
結構與 N-S 相同，但對象從瞬時速度 v 換成時間平均速度 v̄。計算量大幅降低。
ρ (v̄·∇)v̄ — 平均速度對流
平均流場攜帶動量。瞬時擾動的交叉乘積 ρv'v'（Reynolds stress）從這裡被移到右邊，用湍流黏度 μt 來模擬。
−∇p̄ — 時間平均壓力梯度
結構與 N-S 相同，用時間平均壓力 p̄。SIMPLE 演算法的邏輯完全不變。
∇·[(μ+μt)∇v̄] — 有效黏性擴散（最大差異！）
原始 N-S 只有 μ∇²v，RANS 加入了湍流黏度 μt：
• μ：分子黏度（0.011266 Pa·s，燃料鹽本身）
• μt：湍流黏度 = Cμ·k²/ε（k-ε 模型計算）
μt 通常比 μ 大 10~100 倍，湍流大幅增強動量傳遞效率。
⚠️ th.i 目前只有 μ，[FunctorMaterials] 補入 μt 後修正
ρg — 重力（不變）
時間平均不影響重力項，結構完全相同。

k-ε 模型：兩條耦合方程式

RANS 裡的 μt = Cμ·k²/ε 需要 k 和 ε，它們各自有一條輸送方程式，互相依賴，必須聯立求解。

TKE 輸送方程式（求解 k）
ρ ∂k/∂t  +  ρ v·∇k  =  ∇·[(μ+μt/σk)∇k]  +  Pk  −  ρε
ρ ∂k/∂t — TKE 時間項
TKE 隨時間的變化率。注意：沒有 cₚ！
能量方程式需要 ρcₚ ∂T/∂t，因為 T 需要乘 cₚ 才能換算成能量（Q=mcₚΔT）。
但 k 的單位是 J/kg，本身就是能量，所以 ρ ∂k/∂t 就直接是能量密度的時間變化率。
th.i → WCNSFVTurbulentEnergyTimeDerivative，cp=1.0（關掉換算）
ρ v·∇k — TKE 對流
流場把 TKE 從激烈區（近壁面、高剪切區）帶到安靜區。
結構與動量/能量對流完全相同，只是「被帶走的量」從速度/溫度換成了 TKE。
th.i → INSFVTurbulentAdvection
∇·[(μ+μt/σk)∇k] — TKE 擴散
TKE 透過黏性從高 TKE 區擴散到低 TKE 區。
σk = 1.0（Standard k-ε 經驗常數）：TKE 的湍流擴散效率與動量相同。
th.i → INSFVTurbulentDiffusion，sigma=1.0
Pk = μt|∇v|² — TKE 生成項
速度梯度（剪切力）產生湍流。流動越不均勻，剪切越強，TKE 生成越多。
需要 μt → 依賴 ε → 這是兩條方程式耦合的地方！
th.i → INSFVTKESourceSink 的生成部分
ρε — TKE 耗散項
湍流被黏性「磨掉」：大渦旋→小渦旋→分子熱能。
ε 就是 TKED，這裡用到 ε，所以需要 TKED 方程式。
平衡時 Pk = ρε → 充分發展湍流狀態。
th.i → INSFVTKESourceSink 的耗散部分
TKED 輸送方程式（求解 ε）
ρ ∂ε/∂t  +  ρ v·∇ε  =  ∇·[(μ+μt/σε)∇ε]  +  C1(ε/k)Pk  −  C2ρε²/k
ρ ∂ε/∂t
結構與 TKE 完全相同，cp=1.0 同理。
ρ v·∇ε
流場把 TKED 帶走，結構相同。
∇·[(μ+μt/σε)∇ε] — TKED 擴散
σε = 1.3（Standard k-ε 常數），比 σk=1.0 大，
表示 TKED 的湍流擴散比 TKE 稍慢（更難擴散）。
th.i → INSFVTurbulentDiffusion，sigma=1.3
C1(ε/k)Pk — TKED 生成項
C1 = 1.44（經驗常數）。ε/k 的比值把 TKE 生成率換算成 TKED 生成率。
⚠️ 分母含 k → k 不能為零！這是 TKE 初始值設 2.102e-4（非零）的物理原因之一。
C2ρε²/k — TKED 耗散項
C2 = 1.92（經驗常數）。ε²/k 是自我加速耗散。
⚠️ 分母含 k → k=0 時源項→∞，求解器爆掉！
這是 TKE 初始值 絕對不能為零 的根本原因。

為什麼 th.i 需要 [Variables] 和 [FVKernels]？
MOOSE 設計哲學：積木式方程式組合
每個物理機制是一個獨立的 kernel（積木），使用者自由組合。
不同問題只需換不同積木，不用換軟體。這就是為什麼工程師要「手動」宣告變數和方程式。
th.i 區塊對應什麼物理意義
[Variables] 7 個求解未知量 MOOSE 為每個變數分配記憶體，組裝線性系統 Ax=b。每個變數 = 一個求解自由度。
[AuxVariables] power_density 從 OpenMC 傳入的外部量，不是求解目標，是能量方程式的熱源 q‴。這是 Cardinal 耦合的橋樑。
[FVKernels] 25 個方程式項 每個 kernel = 方程式裡的一項。積木式組合成 5 組完整 PDE（偏微分方程式）。
每個 Variable 必須有對應的 FVKernel
