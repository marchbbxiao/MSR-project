「對流」三兄弟：最容易搞混的概念

中文「對流」對應到英文有三個完全不同的單字，出現在三個不同的場合。
方程式裡看到「對流」二字，要先確認是哪一個，否則物理圖像會完全錯位。

⚠️ 本工具中凡出現對流項，均用上標標記區分：
Advection = 流體移動帶走物理量  |  NatConv = 溫差驅動流體運動  |  HeatConv = 固體-流體介面熱交換
Advection  平流／對流輸送
Advection / Convective Transport
物理意義：流體整體移動，順便把某個物理量（動量、熱量、TKE）從 A 帶到 B。

關鍵：是流體本身在移動，帶著量走。

數學形式：ρ v·∇φ（速度場 × 梯度）

出現在：N-S 動量方程式、能量方程式、TKE/TKED 方程式的對流項

MSFR 例子：熔鹽從爐心底部流到頂部，把熱量「帶」上去。
th.i → INSFVMomentumAdvection / INSFVEnergyAdvection / INSFVTurbulentAdvection
NatConv  自然對流
Natural Convection / Buoyancy-driven Flow
物理意義：溫度差 → 密度差 → 浮力 → 驅動流體自發運動。

關鍵：是溫差「主動」引發流動，不需要外力。

數學形式：ρg 項（重力）+ Boussinesq 近似 ρ ≈ ρ₀(1−βΔT)

出現在：動量方程式的重力項（ρgz），配合密度隨溫度變化

MSFR 例子：爐心高溫區燃料鹽密度低 → 往上浮，形成自然循環。
th.i → INSFVMomentumGravity（驅動力來源）+ 密度隨溫度變化
HeatConv  熱對流（熱傳學）
Heat Convection / Convective Heat Transfer
物理意義：固體表面與流體之間的熱量交換，用對流熱傳係數 h 描述。

關鍵：是「介面」的概念，描述壁面和流體之間的熱阻。

數學形式：q = h·A·ΔT（牛頓冷卻定律）

出現在：[FVBCs] 邊界條件，不在體積方程式（FVKernels）裡

MSFR 例子：爐壁把熱量傳給冷卻鹽，h 描述這個介面的熱傳效率。
th.i → [FVBCs]（下一步），不在目前的 FVKernels 裡

Navier-Stokes 動量方程式：完整逐項講解

N-S 方程式建立於 1822 年，是描述所有黏性流體運動的基本定律。地位等同流體力學的 F=ma。

▶ x 方向完整動量方程式（點擊每一項查看詳細說明）
ρ ∂vₓ/∂t  +  ρ (v·∇)vₓ  =  −∂p/∂x  +  μ ∇²vₓ  +  ρgₓ
ρ ∂vₓ/∂t  時間加速度項（慣性項）
物理意義：這塊流體在 x 方向的動量，每秒鐘變化多少。

對應牛頓第二定律 F = ma：
• m（質量）→ ρ（單位體積質量，即密度）
• a（加速度）→ ∂vₓ/∂t（速度的時間導數）

單位：kg/m³ × m/s² = N/m³（每立方公尺的力）

穩態時：∂vₓ/∂t = 0，這一項消失。但在 SIMPLE 迭代過程中仍需要它組裝矩陣。
th.i → INSFVMomentumTimeDerivative ⚠️ SIMPLE 穩態模式已停用（time kernel 會導致 "Vector tag TIME" 錯誤）
ρ (v·∇)vₓ  對流項 Advection ↗
物理意義：流體「流動」本身把 x 方向的動量從 A 點帶到 B 點。
▶ 這是「Advection（平流）」：流體移動帶走動量。不是自然對流，不是壁面熱交換。

展開寫：ρ(vₓ ∂vₓ/∂x + vy ∂vₓ/∂y + vz ∂vₓ/∂z)

為何重要：這是 N-S 方程式唯一的非線性項（速度 × 速度的梯度）。非線性導致：
• 方程式無法解析求解（數學上尚未證明解的存在性！）
• 數值求解需要迭代
• Re 越高，此項相對黏性項越重要 → 湍流 dominates

單位：kg/m³ × (m/s)(m/s)/m = N/m³
th.i → INSFVMomentumAdvection（advected_interp_method=upwind, velocity_interp_method=rc — 在 [GlobalParams] 全域設定）
−∂p/∂x  壓力梯度項
物理意義：壓力差驅動流體從高壓流向低壓（負號代表「壓力降低的方向就是流動方向」）。

這項為什麼關鍵？
SIMPLE 演算法的核心邏輯就在這裡：
1. 假設一個速度場
2. 發現 ∇·v ≠ 0（質量不守恆）
3. 計算壓力修正量 p'
4. 用 −∇p' 修正速度（動量方程式有 −∇p，所以改 p 就能改 v）
5. 迭代直到 ∇·v = 0

這就是為什麼質量守恆方程式的 variable 是 pressure，不是速度！
th.i → INSFVMomentumPressure（pressure=pressure）
μ ∇²vₓ  黏性擴散項
物理意義：分子黏性把動量從快速流體「摩擦傳遞」到慢速流體，就像內部摩擦。

展開：μ(∂²vₓ/∂x² + ∂²vₓ/∂y² + ∂²vₓ/∂z²)

為何 Re=958,000 需要 ~10¹³ 個網格點？詳細推導

第一步：計算 MSFR 的雷諾數

雷諾數 Re 是慣性力與黏性力的比值，判斷流態：

Re = ρ·u·L / μ
密度 ρ
4147.3 kg/m³
平均流速 u = Q/A = 4.5 / (π × 1.1²)
1.184 m/s
特徵長度 L（爐心直徑）
2.2 m
動力黏度 μ
0.011266 Pa·s
Re = 4147.3 × 1.184 × 2.2 / 0.011266
≈ 958,000
⚠️ Re > 4000 = 湍流。MSFR 的 Re 是湍流門檻的 240 倍，是超強湍流狀態。

第二步：DNS 的網格需求估算

DNS（直接數值模擬）必須解析所有尺度的湍流渦旋，從最大渦旋到最小渦旋（Kolmogorov 尺度）。

關鍵：Kolmogorov 微尺度 η

最小渦旋的尺度決定了網格必須多小：

η = L · Re^(−3/4)
特徵長度 L
2.2 m
Re^(3/4) = 958000^0.75
≈ 38,500
η = 2.2 / 38500
≈ 5.7 × 10⁻⁵ m = 57 μm

這意味著網格間距必須小於 57 μm 才能解析最小渦旋。

第三步：估算總網格數

N = (L / η)³ = Re^(9/4)
每個方向的網格數 = L/η = 38,500
≈ 3.85 × 10⁴
三維總網格數 = (3.85×10⁴)³
≈ 5.7 × 10¹³
或用公式 Re^(9/4) = 958000^2.25
≈ 10¹³

第四步：計算資源比較

方法網格數每時間步計算量實際可行性
DNS~10¹³極大❌ 不可行（超越最快超算）
LES~10⁸～10¹⁰大⚠️ 勉強，需超算數週
RANS (k-ε)~10⁵～10⁷小✅ 工程站數小時可完成
💡 RANS 把湍流效果「打包」進兩個方程式（TKE + TKED），用 μt 代替無數個渦旋的效果，讓計算量從 10¹³ 降到 10⁵～10⁷，降低 6~8 個數量級。

物理直覺：湍流的多尺度結構

1
大渦旋（能量注入尺度）
尺度 ~L = 2.2 m（爐心大小）。主流剪切產生，含有大部分湍流動能（TKE）。
2
慣性子區（能量串聯）
大渦旋分裂為中等渦旋，再分裂為小渦旋。這個「能量串聯」過程中能量從大尺度流向小尺度。
3
Kolmogorov 微尺度（耗散）
最小渦旋 η ≈ 57 μm，黏性力終於夠大，把動能轉換為熱能。這就是 TKED（ε）描述的過程。

RANS 完整推導：Reynolds 時間平均法

Step 1：Reynolds 分解（速度拆分）

把每個物理量拆成「時間平均值」+「瞬時擾動值」：

vᵢ = v̄ᵢ + v'ᵢ     p = p̄ + p'     T = T̄ + T'
時間平均的性質：  v̄' = 0（擾動的平均為零）   v̄·w̄ = v̄·w̄（平均的乘積等於乘積的平均—若兩者不相關）

Step 2：代入連續方程式

代入
∂(v̄ᵢ+v'ᵢ)/∂xᵢ = 0
取時間平均
∂v̄ᵢ/∂xᵢ + ∂v'ᵢ/∂xᵢ = 0
因為 v̄'ᵢ = 0
∂v̄ᵢ/∂xᵢ = 0  且  ∂v'ᵢ/∂xᵢ = 0

結論：平均速度和擾動速度各自都滿足連續方程式。

Step 3：代入動量方程式，取時間平均

原始 N-S（代入分解值）
ρ ∂(v̄ᵢ+v'ᵢ)/∂t + ρ(v̄ⱼ+v'ⱼ) ∂(v̄ᵢ+v'ᵢ)/∂xⱼ = −∂(p̄+p')/∂xᵢ + μ∂²(v̄ᵢ+v'ᵢ)/∂xⱼ²
展開對流項（關鍵步驟）
v̄ⱼ·v̄ᵢ + v̄ⱼ·v'ᵢ + v'ⱼ·v̄ᵢ + v'ⱼ·v'ᵢ
取時間平均（擾動的線性項消失，但二次項不為零！）
v̄ⱼ·v̄ᵢ + 0 + 0 + v'ⱼ·v'ᵢ ← 這個不為零！
整理後得到 RANS 方程式
ρ ∂v̄ᵢ/∂t + ρ v̄ⱼ ∂v̄ᵢ/∂xⱼ = −∂p̄/∂xᵢ + μ∂²v̄ᵢ/∂xⱼ² − ρ ∂(v'ᵢv'ⱼ)/∂xⱼ
⚠️ 關鍵！多出了一個新項：−ρ v'ᵢv'ⱼ，這就是 Reynolds Stress Tensor（雷諾應力張量）。
它代表湍流擾動對平均流場的影響，是整個 RANS 理論的核心問題。

Step 4：雷諾應力的封閉問題

Reynolds stress −ρ v'ᵢv'ⱼ 是 6 個未知量（對稱張量），但我們沒有直接的方程式來求解它們。這就是 湍流封閉問題（Closure Problem）。

Boussinesq 假設（1877）：用湍流黏度 μt 來模擬 Reynolds stress：

−ρ v'ᵢv'ⱼ ≈ μt (∂v̄ᵢ/∂xⱼ + ∂v̄ⱼ/∂xᵢ) − (2/3)ρk δᵢⱼ
💡 類比：分子黏度 μ 描述分子尺度的動量傳遞，湍流黏度 μt 描述渦旋尺度的動量傳遞。兩者物理機制不同，但數學形式相同。

Step 5：代入 Boussinesq 假設，得到最終 RANS 方程式

▶ 最終 RANS 動量方程式（點擊比較與 N-S 的差異）
ρ ∂v̄/∂t  +  ρ (v̄·∇)v̄  =  −∇p̄  +  ∇·[(μ+μt)∇v̄]  +  ρg
ρ ∂v̄/∂t — 平均速度時間項
與 N-S 相同，只是對象從瞬時速度 v 換成時間平均速度 v̄。計算量降低：不需要解析每個瞬間的湍流擾動。
ρ (v̄·∇)v̄ — 平均速度對流
對流項的形式不變，但用的是平均速度 v̄。原來包含的 v'v' 交叉項已被移到右邊（Reynolds stress），並用 Boussinesq 假設打包進 μt。
−∇p̄ — 時間平均壓力梯度
結構完全相同，SIMPLE 演算法的邏輯不變。
∇·[(μ+μt)∇v̄] — 有效黏性擴散（⭐ N-S 和 RANS 最大差異）
原始 N-S：μ∇²v（只有分子黏度）
RANS：∇·[(μ+μt)∇v̄]（分子黏度 + 湍流黏度）

μt = Cμ·k²/ε（由 k-ε 模型計算）

在 MSFR 條件下：μt/μ ≈ 10~100，湍流大幅增強動量傳遞。
th.i → INSFVMomentumDiffusion(mu=μ) + INSFVMomentumDiffusion(mu='mu_t', complete_expansion=no) — 兩個獨立 kernel ✅ 已完成：μt 由 AuxVariable mu_t + kEpsilonViscosityAux 計算，μt/μ ≈ 15 倍
ρg — 重力（不變）
時間平均不影響常數項。

Step 6：μt 從哪裡來？→ 需要 k-ε 兩條方程式

μt = Cμ · k² / ε     Cμ = 0.09

k（TKE）和 ε（TKED）各自需要一條輸送方程式 → 這就是下一步的 k-ε 模型。

k-ε 模型：兩條耦合的輸送方程式

RANS 需要 μt，μt 需要 k 和 ε，它們各自滿足一條對流擴散方程式。

TKE 輸送方程式（求解 k）

▶ Turbulent Kinetic Energy（點擊各項）
ρ ∂k/∂t  +  ρ v̄·∇k  =  ∇·[(μ+μt/σk)∇k]  +  Pk  −  ρε
ρ ∂k/∂t — TKE 時間項（無 cₚ！）
為何沒有 cₚ？
能量方程式：ρcₚ ∂T/∂t，因為 T 是溫度，需要乘 cₚ 換算成能量（Q=mcₚΔT）。
TKE 方程式：ρ ∂k/∂t，因為 k 的定義就是 k = ½v'²，單位是 m²/s² = J/kg，本身已是單位質量能量。
所以 ρ·k 就已經是能量密度（J/m³）了，不需要 cₚ 再換算。
th.i → INSFVEnergyTimeDerivative（cp=1.0 關掉換算）⚠️ SIMPLE 穩態已停用 | WCNSFVTurbulentEnergyTimeDerivative 在此版本不存在
ρ v̄·∇k — TKE 對流 Advection ↗
平均流場把 TKE 從高 TKE 區（近壁面、高剪切）帶到低 TKE 區。
▶ Advection：流體移動順便帶走 TKE，與動量/能量的對流項結構完全相同。
th.i → INSFVTurbulentAdvection
∇·[(μ+μt/σk)∇k] — TKE 擴散
TKE 透過黏性從高 TKE 區擴散到低 TKE 區。
σk = 1.0（Standard k-ε 常數）：TKE 的湍流擴散普朗特數，表示 TKE 擴散效率與動量湍流擴散相同。
th.i → INSFVTurbulentDiffusion，sigma=1.0
Pk = μt|∇v̄|² — TKE 生成項
速度梯度（剪切力）產生湍流動能。
• 爐心中流速不均勻的地方（近壁面、入/出口）剪切最強，Pk 最大。
• Pk 依賴 μt，μt 又依賴 k 和 ε → 方程式之間的耦合！
th.i → INSFVTKESourceSink（生成部分，mu_t='mu_t', epsilon=TKED）
ρε — TKE 耗散項（連結到 TKED 方程式！）
湍流動能被黏性「磨掉」：大渦旋→小渦旋→Kolmogorov 尺度→分子熱能。
ε（TKED）就是這個耗散率，它的值由 TKED 方程式求解。
→ 這就是為什麼需要兩條方程式：TKE 方程式用到 ε，TKED 方程式用到 k，互相耦合！
th.i → INSFVTKESourceSink（耗散部分）

TKED 輸送方程式（求解 ε）

▶ Turbulent Kinetic Energy Dissipation Rate（點擊各項）
ρ ∂ε/∂t  +  ρ v̄·∇ε  =  ∇·[(μ+μt/σε)∇ε]  +  C₁(ε/k)Pk  −  C₂ρε²/k
ρ ∂ε/∂t — TKED 時間項（同樣無 cₚ）
ε 的單位是 m²/s³ = W/kg = J/(kg·s)，是「功率密度除以密度」。結構與 TKE 相同，cp=1.0 同理。
ρ v̄·∇ε — TKED 對流 Advection ↗
平均流場把 TKED 從高耗散區帶走。結構相同。th.i → INSFVTurbulentAdvection
∇·[(μ+μt/σε)∇ε] — TKED 擴散
σε = 1.3 > σk = 1.0：TKED 的湍流擴散效率比 TKE 稍低（更難擴散）。
這個差異來自實驗觀測，反映耗散率比 TKE 更「局部」的特性。
th.i → INSFVTurbulentDiffusion，sigma=1.3
C₁(ε/k)Pk — TKED 生成項
C₁ = 1.44（經驗常數）。ε/k 是「每單位 TKE 的耗散率」，把 TKE 生成轉換成 TKED 生成。
⚠️ 分母含 k！→ k=0 時此項趨向無限大，求解器爆掉 → TKE 初始值必須非零！
C₂ρε²/k — TKED 耗散項
C₂ = 1.92（經驗常數）。ε²/k 代表耗散率的自我加速消退。
⚠️ 分母含 k！→ 同上，TKE 初始值 2.102e-4 非零的根本原因之一。

推導鏈完整摘要

1
N-S 方程式（精確）
描述真實瞬時流場。Re=958,000 → DNS 需 10¹³ 網格，無法求解。
2
Reynolds 分解 v = v̄ + v'
拆成平均 + 擾動，對 N-S 取時間平均。擾動的線性項消失，但二次項 v'v' 不為零。
3
Reynolds Stress 出現
多出未知項 −ρv'ᵢv'ⱼ（6個未知量），方程式不封閉。需要湍流封閉模型。
4
Boussinesq 假設
−ρv'ᵢv'ⱼ ≈ μt·∇v̄，把 6 個未知量化成 1 個標量 μt。問題變成：如何計算 μt？
5
k-ε 模型：μt = Cμ k²/ε
引入 TKE(k) 和 TKED(ε) 兩個新變數，各自滿足對流擴散方程式，聯立求解。
6
MOOSE 離散化 → th.i
每條方程式每一項 = 一個 FVKernel。7個 Variables × 對應方程式 = 26個 kernels。
7
Cardinal 耦合
th.i 接收 OpenMC 的 q‴，回傳 T(r)、v(r) 給 OpenMC 更新截面，Picard 迭代直到收斂。
